FAQ

The short answers.

Does the AI ever change a signal by itself?
No. No code path executes without an operator approving a hash-identified plan. The approval requirement is a constant in the code.
What if a change makes things worse?
5-minute monitoring window; ≥20% degradation proposes auto-rollback; any executed plan reverts in one action.
Can the audit log be edited?
Each entry hashes the previous one. Edits break the chain visibly; the console verifies continuously; full JSONL export.
What does it take to run?
One process. Python stdlib + SQLite + a browser. No external database, broker or fleet. Survives restarts intact.
What about privacy?
Edge redaction enforced at ingestion; frames kept only as incident evidence on a retention schedule; no face or plate recognition anywhere.
Can a third party verify a plan?
Yes — a stateless API returns per-rule MUTCD results and verifies any cert's signature + chain membership.
Not in Seattle?
Cities plug in through an adapter (Tacoma is the working example). Start in Shadow against your real feeds.
